## 개념

데이터를 송수신하는 기기들 간의 연결을 통해 정보와 자원을 공유할 수 있게 하는 시스템

## 네트워크 기초 요소들

- 인터넷
    - 여러 컴퓨터와 장치가 서로 연결되어 데이터를 주고 받는 구조
    - 데이터는 일정한 단위로 나뉘어 전송되며 각 노드들은 라우터나 스위치를 통해서 통신함
- 패킷
    - 네트워크를 통해 데이터를 보낼 때 작은 조각 단위로 분할한 데이터의 단위
    - 헤더와 페이로드로 구성 되어 있음
        - 헤더 : 출발 및 목적지 IP,순서,프로토콜 정보로 구성
        - 페이로드: 데이터의 실제 내용
    - 통신 도중 손실되거나 손상되는 패킷들은 TCP 연결이 재전송등을 통해 안전하게 통신하도록 도와줌
- 스위치
    - LAN(근거리 통신)에서 목적지로 데이터 패킷을 전달하는 역할을 함
    - MAC 주소를 기준으로 통신함
    - 브로드캐스트의 범위를 줄여 효율적인 데이터 전달이 가능함
- 라우터
    - 여러 네트워크 간의 통신을 담당하는 역할
    - IP주소를 기반으로 통신함
    - 스위치가 네트워크 내부 통신 (L2)라면
    - 라우터는 네트워크 간 통신 (L3)
- 라우팅 테이블
    - 라우터 내부에 있는 어떤 목적지로 가려면 어떤 경로를 거쳐야 하는지에 대한 정보들이 들어있는 테이블
    - 테이블 구성
        - 목적지 네트워크
        - 서브넷 마스크
        - 게이트웨이(IP주소)
        - 인터페이스(어떤 포트로 나갈지)
    - 동적 라우팅을 통해 자동 갱신 가능
- IP주소
    - 네트워크 간의 유일하게 가지고 있는 고유 주소
    - IPv4 는 32비트
    - IPv6 는 128비트
    - 공인IP ↔ 사설IP
        - 공인 IP - 다른 네트워크와 통신하기 위한 대표 IP(유일)
        - 사설 IP - 네트워크 내부의 각각이 가진 내부 IP(NAT를 통해 통신)
- Mac 주소
    - 네트워크 장치에 존재하는 물리적 네트워크 주소
    - IP주소는 변경가능하지만 MAC주소는 변경 불가능

## Layered 구조

시스템을 구조화하여 역할을 분리하고 독립적인 구조이기 때문에 문제가 생기고 변경을 해야할 때 다른 계층에 최소한의 영향을 끼칠 수 있는 특징이 있다. 계층 간 분리되어있기 때문에 확장성이 높다. 체계적으로 시스템을 관리하고 유지보수를 쉽게 하기 위한 구조

### OSI 7계층 구조

네트워크 계층을 7가지 단계로 나눈 구조

- 1계층 물리 계층(Physical Layer)
    - 물리적 장치간 데이터 전송
- 2계층 데이터 링크 계층(Data Link Layer)
    - 데이터 전송 오류 관리 및 링크 관리
- 3계층 네트워크 계층(Network Layer)
    - 데이터 경로 설정 및 라우팅
- 4계층 전송 계층(TransPort Layer)
    - 데이터 전송의 신뢰성 보장
- 5계층 세션 계층(Session Layer)
    - 세션 설정 관리 및 종료
- 6계층 표현 계층(Presentation Layer)
    - 데이터 변환 및 암호화
- 7계층 응용 계층(Application Layer)
    - 사용자와 어플리케이션 간의 인터페이스 제공

### TCP/IP 4계층 구조

인터넷 프로토콜을 기반으로 한 계층 구조

- 1계층 네트워크 액세스
    - 물리적 네트워크 연결
- 2계층 인터넷 계층
    - 패킷 전달 및 경로 설정
- 3계층 전송 계층
    - 데이터 전송 및 신뢰성 보장
- 4계층 응용 계층
    - 사용자 서비스 및 애플리케이션 프로세스
